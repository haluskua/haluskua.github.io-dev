Difference between Html and Css

HTML

which stands for hyper Text Markup Languange

I look as html as the body container for elements to sit in. One way I try to imagine is that Html is like a house. A house has ojects and properties such as window, wall, roof, floor and table etc. Html is the make up of a house and what is in it.

CSS

which stands for Cascading style sheets

When compared with Html, Css basically allows for Html to be styled, and be organized respectively so that it can be viewd well on different devices. Think of css as the builder or handyman that can add definition or value to the properties such as paint the wall or move the table from the lounge to the kitchen etc.

Javascripth

Dynamic moderator

Well Javascripth really moves the house. With Javascripth, a webpage can become more dynamic in terms of interaction. A house with Javascripth would probably have a door bell or an automatic garage door or sensor light on the porch etc. This improves interaction with occupants and makes living a little more interesting. The same applies to webpages. Websites with Javascripth are more interactive, and playfull instead of being static.

Explain what Functions are and why they are useful?

Functions in programming contains code that can be used over and over again once defined. A function can return a value or a procedure that may not return a value. Specifically, a function contains a unit of code that works on various inputs, many of which are variables, and produces concrete results involving changes to variable values or actual operations based on the inputs. Simply I just see function as a command / rules that forces an outcome. Creating a function helps one solve an issue or make life easier when trying to figure out something.
